### 分类分析 (Classification) —— 预测“高犯罪风险”区域

这个方案的核心思想是：我们能否通过一个地区的城市特征（人口、住房、交通等），准确预测该地区是否属于“高犯罪风险区”？

- **研究问题 (Research Question):** Is it possible to accurately classify London boroughs into high or low crime risk categories using socio-demographic and built environment features? (是否可以利用社会人口和建成环境特征，准确地将伦敦各区分类为高犯罪风险或低犯罪风险类别？)
- **主数据集:** 伦敦犯罪数据（计算总犯罪率）。
- **补充数据集:** 英国人口普查数据（如人口密度、年轻人比例、租房比例等）。
- **如何计算 Rate 并构建目标变量 (y):**
  - 先计算 `总犯罪率 = (总犯罪数 / 常住人口) * 1000`。
  - **特征工程（二值化）:** 将犯罪率排名前 25% 的区域标记为 `1` (High Risk)，其余 75% 的区域标记为 `0` (Low Risk)。这样你就把一个回归问题巧妙地变成了分类问题。
- **推荐的 4 个核心分析步骤 (完美契合大纲要求):**
  1. **数据融合与特征工程:** 合并犯罪率标签与人口普查特征，并检查类别不平衡（Class Imbalance）。
  2. **逻辑回归分类器 (Logistic Regression):** 作为基线模型 (Baseline Model) 进行初步分类，并提取系数，解释哪些社会因素显著增加了高风险概率。
  3. **随机森林分类器 (Random Forest Classifier):** 训练一个更强大的非线性分类器，并提取 **特征重要性 (Feature Importance)**，找出对预测高犯罪率最重要的 3 个因素。
  4. **模型评估与对比 (Model Evaluation):** 使用混淆矩阵 (Confusion Matrix) 和分类报告。
     - *高分亮点：* 引用 Practical 2 中学到的知识，强调在这个任务中，**召回率 (Recall) 比精确率 (Precision) 更重要**，因为漏判一个“高风险区”（False Negative）的代价很高。

### 循序渐进分析计划 (Step-by-Step Analysis Plan)

**步骤 1: 数据融合与特征工程 (Data Fusion and Feature Engineering)**

- 加载 MPS 犯罪数据，按 Ward Code 分组，得出总犯罪数。
- 使用 Ward Code 将此数据与 Ward Profiles 数据集合并。
- **计算比率 (Calculate Rate):** 创建一个新列：`Crime_Rate = (Total_Crime / Population) * 1000`。
- **二值化目标变量 (Binarize Target):** 确定 `Crime_Rate` 的第 75 百分位数。创建你的目标变量 `y`：如果比率 >= 第 75 百分位数，则为 `1`，否则为 `0`。
- *关键防雷指南：* 在建模前，务必从你的特征矩阵 `X` 中删除 `Ward_Code` 和 `Ward_Name`。

**步骤 2: 探索性数据分析 (EDA) 与空间可视化 (EDA and Spatial Visualization)**

- 使用 `geopandas` 将你计算出的犯罪率和风险标签与 ONS Ward 边界合并。
- 绘制一张等值线地图 (choropleth map)，展示二值化的高风险 vs 低风险 wards。这将在视觉上锚定你的项目，并满足评分标准中对可视化的要求。

**步骤 3: 基线建模 (逻辑回归) (Baseline Modeling (Logistic Regression))**

- 执行训练-测试集划分 (train-test split)（例如 80/20），确保使用 `stratify=y` 来保持两组数据中 1:3 的类别比例不变。
- 使用 `StandardScaler` 对特征进行缩放。
- 拟合一个逻辑回归模型。提取并可视化系数 (coefficients)，以解释特征的方向性关联（例如，“较高的 IMD 分数与一个 ward 被分类为高风险呈正相关”）。

**步骤 4: 进阶建模 (随机森林) (Advanced Modeling (Random Forest))**

- 拟合一个 `RandomForestClassifier`。显式设置 `class_weight='balanced'`，以惩罚模型漏报少数类（高风险 wards）的行为。
- 提取 `feature_importances_`，并绘制条形图，展示驱动这种分类的前 5-10 个最重要特征。

**步骤 5: 模型评估与比较 (Model Evaluation and Comparison)**

- 在测试集上生成两个模型的预测结果。
- 输出两者的 `confusion_matrix`（混淆矩阵）和 `classification_report`（分类报告）。
- 重点关注 **Recall (召回率)** 来比较模型。解释在资源分配场景（如治安或消防服务）中，漏报（False Negative，未能识别出高风险 ward）比误报（False Positive）的危害更大。
